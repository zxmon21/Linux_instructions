QNAP Container Infrastructure

Documentation for containerized services, storage locations, networking, reverse proxy access, and external authentication setup on the QNAP NAS.

Infrastructure Overview We have a QNAP TS-453D NAS at home that serves as a storage and application server.
  • Our NAS is running at 192.168.50.140.
  • The NAS is configured to be reachable under a domain name. See NAS -> MyQNAPcloudLink -> DDNS
  • You can access it via SSH on port 2218 inside and outside the home network. Control panel -> Network -> SSH. Account admin is disabled for security reasons.
  • The NAS is providing SSL certs through QNAP onboard configuration of LetsEncrypt. Control panel -> System -> Security -> SSL Certificate & Private key -> Server certificate
  • The NAS is running Container Station for Docker management. It is hosting these applications with containers:
    • Minecraft server
    • Authelia for authentication of web services external access
    • Paperless for document management
    • YT-DLP web ui (internal only)
    • RUT
