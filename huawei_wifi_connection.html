Huawei SUN2000 WiFi (AP Mode) Setup on Ubuntu

  1. Install required tools 
    sudo apt update
sudo apt install network-manager netcat-openbsd
  2. Identify your WiFi interface 
    nmcli device status

    Look for something like wlp2s0 (USB WiFi adapter).

  3. Scan for Huawei inverter WiFi 
    nmcli device wifi list

    Look for SSID like SUN2000-HV2290249916.

  4. Create a password file for WiFi authentication 
    nano ~/sun2000_passwd.conf

    Add the WiFi PSK (Huawei default is often Changeme unless changed): 

    802-11-wireless-security.psk:Changeme

    Secure the file:

    chmod 600 ~/sun2000_passwd.conf
  5. Create WiFi connection profile with WPA2 security (single step) 
    sudo nmcli connection add \
type wifi \
ifname wlp2s0 \
con-name sun2000 \
ssid "SUN2000-HV2290249916" \
wifi-sec.key-mgmt wpa-psk
  6. Activate connection using passwd-file 
    sudo nmcli connection up sun2000 \
passwd-file ~/sun2000_passwd.conf
  7. Create a stable static IP configuration (recommended) 
    sudo nmcli connection modify "SUN2000-HV2290249916" \
ipv4.method manual \
ipv4.addresses 192.168.200.2/24 \
ipv4.gateway 192.168.200.1 \
ipv4.dns 192.168.200.1

    This avoids unreliable DHCP from the inverter.

  8. Reconnect using the new configuration 
    sudo nmcli connection down "SUN2000-HV2290249916"
sudo nmcli connection up "SUN2000-HV2290249916"
  9. Verify IP configuration 
    nmcli device show wlp2s0
ip route

    You should see:
    192.168.200.2/24 and route to 192.168.200.0/24

  10. Test Modbus connectivity 
    nc -zv 192.168.200.1 6607

    If successful, the inverter is reachable.

  11. Create WiFi recovery script 
    nano ~/restore_sun.sh

    Paste:

    #!/bin/bash

IF="wlp2s0"
SSID="SUN2000-HV2290249916"

nmcli device disconnect "$IF" 2>/dev/null
pkill -f "dhclient.*$IF" 2>/dev/null

ip link set "$IF" down
sleep 3
ip link set "$IF" up
sleep 5

nmcli device wifi connect "$SSID" ifname "$IF"
  12. Make script executable 
    chmod +x ~/restore_sun.sh
  13. Create watchdog script 
    nano ~/sun_watchdog.sh

    Paste:

    #!/bin/bash

if ! nc -z -w3 192.168.200.1 6607; then
    ~/restore_sun.sh
fi
  14. Make watchdog executable 
    chmod +x ~/sun_watchdog.sh
  15. Run watchdog automatically (cron) 
    crontab -e

    Add:

    */2 * * * * /home/$USER/sun_watchdog.sh
  16. Verify everything works 
    nc -zv 192.168.200.1 6607
nmcli device status

    If WiFi drops, it should recover automatically within ~2 minutes.

  17. Notes

    - Huawei SUN2000 WiFi AP mode is unstable for long-term use
    - Static IP improves reliability significantly
    - Ethernet or WiFi client mode is more stable if available
